# Native biometric vault platform gotchas

- **iOS target membership is explicit for `NativeBiometricVault`.** This repo's `ios/EnboxMobile.xcodeproj/project.pbxproj` carries belt-and-suspenders `PBXBuildFile` / `PBXFileReference` / `PBXSourcesBuildPhase` entries for `RCTNativeBiometricVault.{h,mm}`. Adding a new Obj-C++ file under `ios/EnboxMobile/NativeBiometricVault/` is not enough by itself; verify the `.mm` file appears in the target sources or the TurboModule will not be compiled into the app binary. The sibling `NativeSecureStorage` and `NativeCrypto` modules are **not** wired through pbxproj — they rely solely on `use_native_modules!` + `codegenConfig.ios.modulesProvider` in `package.json`. See `.factory/library/ios-target-membership.md` for the full rationale and invariants.
- **Android Keystore AES/GCM must generate its own IV when randomized encryption is required.** If a key is created with `setRandomizedEncryptionRequired(true)`, initialize encryption with `cipher.init(Cipher.ENCRYPT_MODE, key)` and persist `cipher.iv`. Supplying a caller-generated IV through `GCMParameterSpec` can fail for Keystore-backed keys during encryption.
